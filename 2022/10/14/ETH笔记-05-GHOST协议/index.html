ETH-GHOST协议

B站视频链接

以太坊的共识机制：GHOST 协议

与比特币的对比

比特币的出块时间设计为为 $10$ 分钟，而以太坊的出块时间大大减少，为 $12-14$ 秒

如果使用原先的共识机制，将会产生大量的临时性分叉，并且导致不公平的出现（mining centralization），大型矿池所挖出的块更有可能称为最长合法链（centralization bias）

以太坊使用基于 GHOST 的协议，其对于矿工挖出的 orphan block（以太坊中称为 uncle block）也会给与一定奖励

GHOST 协议及发展

最初版本的 GHOST 协议规则如下：

graph RL
    A1[1]
    A2[2]-->A1
    A3[3]-->A2
    A4[4]-->A3
    A5[5]-->A3
    A6[6]-->A3
    A7[7]-->A5
    A8[8]-->A3
    style A4 fill: #395260
    style A6 fill: #395260

这时 $7$ 所在的链成为最长合法链，$4$、$6$ 和 $8$ 是 uncle block

$7$ 块可以将 $4$ 和 $6$ 包含在区块里，被包含 uncle block 的挖出矿工可以获得 $\frac{7}{8}$ 的出块奖励，而包含一个 uncle block 可以额外获得 $\frac{1}{32}$ 的出块奖励

一个区块最多可以包含 $2$ 个 uncle block

这样会导致，如上图如果 $7$ 包含了 $4$ 和 $6$，将不能再包含 $8$，这对 $8$ 是不公平的

同时也可能出现由于矿池间竞争，$7$ 故意不包含 $4$ 和 $6$ 等

GHOST 协议修改版，不再“区分辈分”，即在如下情况中：

graph RL
    A1[1]
    A2[2]-->A1
    A3[3]-->A2
    A4[4]-->A3
    A5[5]-->A3
    A6[6]-->A3
    A7[7]-->A5
    A8[8]-->A3
    A9[9]-->A7
    style A4 fill: #395260
    style A6 fill: #395260

$4$、$6$ 和 $8$ 也是 $9$ 的 uncle block

如果 $7$ 未将 $8$ 包含，那么 $9$ 仍可将 $8$ 当作 uncle block 包含

这样导致一个问题，新挖出的区块可能会包含一个非常久远的叔父区块，从而导致其他问题，需要对 GHOST 协议再做修改

GHOST 协议修改版 $2.0$，限制更久远的叔父区块获得的奖励，如下：

graph RL
    A1[1]
    A2[2]-->A1
    A3[3]-->A2
    A4[4]-->A3
    A5[5]-->A4
    A6[6]-->A5
    A12[2']-->A1
    A22[3']-->A2
    A32[4']-->A3
    A42[5']-->A4

若 $6$ 包含它真正的 uncle block $5’$，那么 $5’$ 可以获得 $\frac{7}{8}$ 的出块奖励

若包含两代前的 uncle block $4’$，$4’$ 可以获得 $\frac{6}{8}$ 的出块奖励

若包含三代前的 uncle block $3’$，$3’$ 可以获得 $\frac{5}{8}$ 的出块奖励

以此类推，最久可以包含六代前的 uncle block，其获得 $\frac{2}{8}$ 的出块奖励

而更久远的 uncle block 将不能被包含（七代前的叔父区块并非是 $\frac{1}{8}$ 的出块奖励，而是不能被包含）

即，包含的 uncle block 和当前区块在七代内需要有共同祖先

对于新挖出的区块，包含一个叔父区块的奖励仍是 $\frac{1}{32}$，最多仍是包含 $2$ 个叔父区块

这种设计可以鼓励出现分叉后尽快进行包含合并，以解决临时性分叉的问题

注意：只有出现后的第一个区块可以得到 uncle reward，之后的区块将不能得到奖励

区块与奖励

以太坊中挖出的区块的矿工可以获得两部分奖励：出块奖励 block reward 和 区块中包含的智能合约执行所付出的汽油费 gas fee（类似于比特币的 tx fee）

uncle block 只能获得 block reward，不能获得 gas fee

当新区块包含一个 uncle block 时，仅作一部分合法性检查，uncle block 中包含的交易不会被执行
